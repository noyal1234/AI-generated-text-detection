## Overview

The AI-Generated-Text-Detection project is designed to identify whether a given piece of text was generated by an AI language model (like GPT, ChatGPT, etc.) or written by a human. Using a fine-tuned BERT model trained on diverse datasets, this application achieves high accuracy in distinguishing between human-written and AI-generated content.

This tool is valuable for:
- Academic institutions detecting AI-assisted assignments
- Content platforms identifying synthetic content
- Researchers studying AI language models
- Organizations ensuring content authenticity
- Writers protecting against plagiarism detection false positives

## Features

- **BERT-Based Detection**: Leverages the power of BERT (Bidirectional Encoder Representations from Transformers) for accurate text analysis
- **Real-Time Inference**: Fast detection with minimal latency on CPU and GPU
- **Web Interface**: User-friendly Flask-based web application for easy access
- **Probability Scores**: Returns confidence scores indicating the likelihood of AI generation
- **Batch Processing**: Support for processing multiple texts efficiently
- **Fine-Tuned Model**: Pre-trained model optimized for AI-generated text detection
- **REST API**: JSON-based API for integration with other applications
- **Responsive Design**: Mobile-friendly web interface for all devices

## How It Works

### Detection Pipeline

1. **Text Input**: User submits text through the web interface or API
2. **Tokenization**: Text is tokenized using BERT tokenizer (max 512 tokens)
3. **Model Inference**: Tokenized text passes through the fine-tuned BERT model
4. **Classification**: Model outputs probability scores for two classes:
   - Class 0: Human-written text
   - Class 1: AI-generated text
5. **Result Display**: Probability of AI generation is returned to the user

### Model Architecture

```
Input Text
    ↓
Tokenization (BertTokenizer)
    ↓
BERT Encoder
    ├── Token Embeddings
    ├── Positional Embeddings
    └── Segment Embeddings
    ↓
Self-Attention Layers (12 layers)
    ↓
Feed-Forward Networks
    ↓
Classification Head
    ├── Dropout
    └── Linear Layer (2 classes)
    ↓
Softmax
    ↓
Output Probability
```

## Usage

### Starting the Application

```bash
python app.py
```

The application will start on `http://localhost:5000`

### Web Interface

1. Open your browser and navigate to `http://localhost:5000`
2. Enter or paste the text you want to analyze
3. Click the "Detect" button
4. View the results showing the AI generation probability

### Using the API

#### Detect AI-Generated Text

**Endpoint:** `POST /predict`

**Request:**
```bash
curl -X POST http://localhost:5000/predict \
  -H "Content-Type: application/json" \
  -d '{"text": "Your text here"}'
```

**Request Body:**
```json
{
  "text": "The quick brown fox jumps over the lazy dog."
}
```

**Response:**
```json
{
  "aiPercentage": 15.45
}
```

**Response Fields:**
- `aiPercentage`: Probability (0-100) that the text is AI-generated
  - 0-30: Likely human-written
  - 30-70: Uncertain/Mixed
  - 70-100: Likely AI-generated

### Error Handling

**Empty Text:**
```bash
curl -X POST http://localhost:5000/predict \
  -H "Content-Type: application/json" \
  -d '{"text": ""}'
```

**Response:**
```json
{
  "error": "Input text is empty"
}
```

**HTTP Status:** 400

## Model Details

### Model Information

- **Base Model**: BERT (Bidirectional Encoder Representations from Transformers)
- **Model Size**: bert-base-uncased
- **Parameters**: ~110 million
- **Vocabulary Size**: 30,522 tokens
- **Max Sequence Length**: 512 tokens
- **Output Classes**: 2 (Human: 0, AI-generated: 1)

### Tokenization

- **Tokenizer**: BertTokenizer
- **Case Handling**: Lowercase (uncased model)
- **Padding**: Dynamic padding to max length
- **Truncation**: Enabled for sequences longer than 512 tokens

### Training Details

The model was fine-tuned on datasets containing:
- Human-written text samples
- AI-generated text samples (GPT-2, GPT-3, etc.)
- Mixed and synthetic content

**Training Parameters** (from FineTune.ipynb):
- Learning Rate: 2e-5
- Batch Size: 16-32
- Epochs: 3-5
- Optimizer: AdamW
- Loss Function: Cross-Entropy

## Training

To fine-tune the model on your own dataset:

### Using Jupyter Notebook

```bash
jupyter notebook FineTune.ipynb
```

Follow the notebook for:
1. Loading and preparing your dataset
2. Preprocessing and tokenization
3. Model fine-tuning
4. Evaluation and validation
5. Saving the trained model

### Dataset Format

Prepare your dataset as a CSV file with two columns:

```csv
text,label
"This is human written text",0
"This text was generated by an AI model",1
```

**Columns:**
- `text`: The text sample
- `label`: 0 for human-written, 1 for AI-generated

### Training Script Example

```python
from transformers import BertTokenizer, BertForSequenceClassification, Trainer, TrainingArguments
from datasets import load_dataset

# Load dataset
dataset = load_dataset('csv', data_files='your_dataset.csv')

# Load tokenizer and model
tokenizer = BertTokenizer.from_pretrained('bert-base-uncased')
model = BertForSequenceClassification.from_pretrained('bert-base-uncased', num_labels=2)

# Tokenize
def tokenize_function(examples):
    return tokenizer(examples['text'], padding=True, truncation=True)

tokenized_datasets = dataset.map(tokenize_function, batched=True)

# Train
training_args = TrainingArguments(
    output_dir='./results',
    num_train_epochs=3,
    per_device_train_batch_size=16,
    learning_rate=2e-5
)

trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=tokenized_datasets['train']
)

trainer.train()
model.save_pretrained('./my_model')
```

### Available Models

- `bert-base-uncased`: 12 layers, 768 hidden units (default)
- `bert-base-cased`: 12 layers, 768 hidden units (case-sensitive)
- `bert-large-uncased`: 24 layers, 1024 hidden units (slower, more accurate)
- `bert-large-cased`: 24 layers, 1024 hidden units (case-sensitive, slower)

## Troubleshooting

### Model File Not Found

**Error:** `FileNotFoundError: my_model.pth`

**Solution:**
1. Ensure `my_model.pth` is in the project root
2. Train a new model using `FineTune.ipynb`
3. Download from releases page

### Out of Memory Error

**Error:** `RuntimeError: CUDA out of memory`

**Solutions:**
- Reduce batch size in training
- Use CPU instead of GPU: `torch.device('cpu')`
- Use a smaller BERT model (bert-base instead of bert-large)
- Close other applications

### Port Already in Use

**Error:** `Address already in use`

**Solution:**
```bash
# Change port in app.py
app.run(debug=True, port=5001)

# Or kill the process
lsof -i :5000
kill -9 <PID>
```

### Slow Inference

**Causes:**
- Using CPU instead of GPU
- Large text inputs
- Model too large

**Solutions:**
- Enable GPU: `torch.device('cuda')`
- Reduce text length
- Use bert-base instead of bert-large
- Add caching for repeated requests

### Tokenizer Not Found

**Error:** `OSError: Can't load tokenizer`

**Solution:**
```bash
# Download tokenizer manually
python -c "from transformers import BertTokenizer; BertTokenizer.from_pretrained('bert-base-uncased')"
```
